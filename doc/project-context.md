## 一、项目定位

**参赛作品**：嵌入式芯片与系统设计竞赛，主题"AI 赋能设计，设计点亮 AI！"

**产品定义**：面向儿童（4–12岁）的智能步态矫正鞋垫系统。通过足底 FSR 压力传感器实时监测步态，检测内八/外八异常，即时反馈纠正，并通过 AI 生成健康报告供家长查看。

**核心价值主张**：
- 低成本硬件（FSR）+ AI 语义增强 = 类专业步态分析能力
- 边缘-云协同 AI 分层：STM32 端 TinyML 实时推理 + 云端 LLM 生成报告
- 完整 AIoT 闭环：采集 → 传输 → 分析 → 反馈

## 二、硬件现状（已打板完成）

| 硬件 | 规格 | 说明 |
|------|------|------|
| 主控 | STM32 自研 PCB | 已设计打板 |
| 压力传感器 | FSR 鞋垫，32 路总计（左脚 16 路 / 右脚 16 路） | 当前为分区峰值，非连续分布 |
| 物联网模组 | LTE + GPS + BLE 一体模组 | 型号暂不涉及，抽象AT指令层 |
| 姿态传感器 | IMU（三轴加速度 + 三轴角速度） | 辅助步态事件检测 |
| 反馈器件 | LED 指示灯 + 震动马达 | 步态异常时实时提醒 |
| 显示 | OLED 小屏幕 | 低优先级 |

**关于 FSR 数据的重要说明**：
FSR 空间分辨率有限，系统不做物理压力场重建，而是通过深度学习进行**步态语义特征提取**（足底压力的高维语义表示）。这是本项目的核心技术亮点表述。

## 三、系统架构（已确认）

```
STM32（鞋垫端）
 ├── BLE Notify（50Hz）──→ 手机 Web（近程实时可视化）
 └── LTE HTTP POST（批量）──→ 公网云后端（存储 + AI 分析）
                                    ↑ HTTPS GET
                             手机 Web（远程查看模式）
```

**关键决策**：
- 手机 Web 是**纯消费端**，不做数据中转
- BLE 和 LTE 是 STM32 的两条独立输出链路
- 手机 Web 有两个模式：近程蓝牙实时可视化 / 远程查看历史报告

## 五、功能清单（优先级排序）

### 必须实现
1. BLE 接收 32 路压力值 → Three.js 足底热力图实时渲染
2. COP（压力中心）轨迹实时计算与展示
3. STM32 TinyML 步态分类（正常/内八/外八）+ 置信度显示
4. LED/震动马达异常实时反馈
5. STM32 经 LTE 直传原始压力帧至后端
6. 后端调用 DeepSeek API 生成每日健康日报
7. 前端远程查看 AI 日报

### 扩展实现（时间充裕时）
- GPS 运动轨迹地图
- 历史步态评分趋势图
- 维稳测试标准化模式
- OLED 显示设备状态
