## 一、项目背景

**面向比赛：嵌入式芯片与系统设计竞赛（主题：AI 赋能设计，设计点亮 AI！）**

儿童期（4–12 岁）是足弓与下肢力线的发育黄金期。内八/外八步态若未及时纠正，长期将导致膝关节、踝关节及脊柱的不可逆损伤。传统矫治依赖医院定期复诊，缺乏居家、实时、量化的连续监测手段。

本项目围绕柔性 FSR（力敏电阻）鞋垫，构建一套**儿童步态健康 AIoT 系统**：

- 鞋垫实时采集足底压力 → BLE 推送手机可视化 → LTE 上传云端 → AI 生成健康报告
- 硬件自研打板（STM32），体现嵌入式硬件落地能力
- 边缘-云协同 AI 分层架构，对应赛题"AI 赋能"

## 二、设计思路

### 2.1 核心架构：端-云-手机三层

```
                         ┌─────────────────────────────────┐
                         │        云后端（Cloud）            │
                         │  HTTP 数据接入 / 原始数据存储     │
                         │  特征提取 / DeepSeek 报告生成     │
                         │  REST API 服务                   │
                         └────────────┬────────────────────┘
                                      │ LTE（STM32 直传）
                                      │              ↑ HTTPS（远程查看）
┌──────────────────────┐              │              │
│   STM32（鞋垫）       │──────────────┘         ┌───┴─────────────────┐
│  32路 FSR 采集        │                        │   手机 Web（Android）│
│  IMU 姿态采集         │                        │                      │
│  基础信号处理         │──── BLE ──────────────→│  模式A：近程蓝牙     │
│  步态事件检测         │   实时压力帧             │   实时热力图渲染     │
│  TinyML 推理         │                        │   COP 轨迹           │
│  LED/马达反馈         │                        │                      │
│  LTE 直传后端         │                        │  模式B：远程查看     │
└──────────────────────┘                        │   历史数据 / AI报告  │
                                                └─────────────────────┘
```

### 2.2 两项关键架构决策

**① 两条独立链路（互不依赖）**

- **BLE 链路**：STM32 → 手机 Chrome，<50ms 延迟，实时可视化专用，不涉及后端
- **LTE 链路**：STM32 → 公网云后端，直传，手机不做中转

**手机 Web 是纯消费端，不参与数据转发**。两种模式可同时使用也可独立使用。

**② AI 分层体系**

| 层级 | 运行位置 | 负责内容 |
|------|---------|---------|
| TinyML（边缘） | STM32 | 步态分类：正常 / 内八 / 外八 |
| LLM（云端） | DeepSeek API | 自然语言健康日报生成 |

TinyML 负责实时性，LLM 负责解释性；职责清晰，避免 "AI 硬塞"。

### 2.3 设计取舍（比赛阶段，从简原则）

| 取舍项 | 决策 | 原因 |
|--------|------|------|
| 用户登录系统 | 无，单设备硬编码 | 只有一台设备，增加复杂度无收益 |
| iOS 兼容 | 不做，仅 Android Chrome | Web Bluetooth 支持良好，演示用安卓机 |
| 离线缓存 | 暂不实现 | 嵌入式部分抽象处理 |
| 标定曲线 | 待后续补充 | 当前只有标定 Demo |
| 足底坐标映射 | 待后续确定 | 留 TBD 占位 |

## 三、功能要点

### 必须实现（比赛核心演示）

| 编号 | 功能 | 说明 |
|------|------|------|
| F1 | 足底压力实时热力图 | BLE 接收 32 路压力值，Three.js 渲染动态热力图 |
| F2 | COP 轨迹展示 | 实时计算压力中心坐标，Canvas/Three.js 绘制轨迹 |
| F3 | 步态状态显示 | 展示 TinyML 分类结果（正常/内八/外八）及置信度 |
| F4 | LED/马达实时反馈 | 步态异常时板载 LED 变色 + 马达震动提醒 |
| F5 | 数据云端存储 | LTE 异步批量上传原始压力数据至云后端 |
| F6 | DeepSeek 日报生成 | 后端汇总当日特征，调用 DeepSeek API 生成健康报告 |
| F7 | 前端报告展示 | 前端页面展示 AI 生成的健康日报 |
