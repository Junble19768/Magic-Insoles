# User

我有一段二维图像上的轮廓线，其中有一些小小抖动，我希望将其做的平滑一些，opencv有现成工具吗

# ChatGPT

有，OpenCV 有几种方法可以对二维轮廓线（contour）进行平滑，但要看你的“抖动”属于哪一类：

* **轮廓点数量很多，局部有小噪声抖动** → 曲线平滑（推荐）
* **轮廓本身有很多锯齿点** → 降采样 + 拟合
* **希望保持几何形状，只去掉小尺度扰动** → 多边形近似

下面按常用程度介绍。

---

## 方法1：`cv2.approxPolyDP()`（最常用，但不是平滑）

OpenCV 自带的轮廓简化：

```python
epsilon = 2.0  # 越大越平滑
approx = cv2.approxPolyDP(contour, epsilon, True)
```

原理是 Douglas-Peucker 算法：

* 删除接近直线的小波动
* 减少轮廓点数量
* 保留整体形状

效果：

```
原始:
~~~~~~/\~~~~~~
     /
平滑:
------/\------
```

优点：

* 简单
* 快
* OpenCV 原生

缺点：

* 它不是连续曲线平滑
* 对圆弧、自由曲线可能产生折线

适合：

> 物体边缘检测后，需要简化轮廓

---

## 方法2：Savitzky-Golay 滤波（推荐用于你的情况）

如果你的轮廓是一串二维点：

```
[(x0,y0),
 (x1,y1),
 ...
 (xn,yn)]
```

可以分别对 x 和 y 做平滑：

```python
from scipy.signal import savgol_filter

import numpy as np

points = contour[:,0,:]

x = points[:,0]
y = points[:,1]

x_smooth = savgol_filter(
    x,
    window_length=11,
    polyorder=3
)

y_smooth = savgol_filter(
    y,
    window_length=11,
    polyorder=3
)

smooth_contour = np.stack(
    [x_smooth,y_smooth],
    axis=1
)
```

效果：

* 保留曲率
* 不会像均值滤波一样变形
* 对小抖动非常有效

参数：

| 参数            | 作用     |
| ------------- | ------ |
| window_length | 平滑窗口大小 |
| polyorder     | 拟合阶数   |

例如：

轻微抖动：

```python
window_length=7
polyorder=2
```

明显抖动：

```python
window_length=21
polyorder=3
```

---

## 方法3：B样条曲线拟合（效果最好）

如果你的轮廓需要作为机器人视觉、轨迹规划等用途，我更推荐这个。

流程：

```
离散轮廓点
      |
      ↓
B-spline拟合
      |
      ↓
连续光滑曲线
```

Python：

```python
from scipy.interpolate import splprep, splev

points = contour[:,0,:]

x = points[:,0]
y = points[:,1]


tck,u = splprep(
    [x,y],
    s=5
)

u_new=np.linspace(
    0,1,500
)

x_new,y_new=splev(
    u_new,
    tck
)

smooth=np.stack(
    [x_new,y_new],
    axis=1
)
```

`s` 控制平滑程度：

小：

```python
s=0
```

接近原始曲线

大：

```python
s=100
```

更加平滑

优点：

* 曲线连续
* 一阶二阶导数连续
* 非常适合：

  * 机器人轨迹
  * 边界重建
  * grasp pose生成
  * CAD轮廓

你的应用场景和普通的“轮廓去噪”有一点区别：你不是为了识别物体，而是希望**建立一个 FSR 阵列的数字化几何模型**，后续用于：

# 2. 轮廓重采样（非常重要）

不要直接拟合。

先让点沿曲线均匀分布。

例如：

原始：

```
***********************
**
 *
  *
   ****************
```

可能某些地方：

```
1 pixel一个点
```

某些地方：

```
10 pixel一个点
```

拟合会受到影响。

推荐：

计算弧长：

[
s_i=\sum ||p_i-p_{i-1}||
]

然后等距离采样：

例如：

```
500个点
```

代码：

```python
import numpy as np


def resample_contour(points, n=500):

    d = np.sqrt(
        np.sum(
            np.diff(points,axis=0)**2,
            axis=1
        )
    )

    s=np.insert(
        np.cumsum(d),
        0,
        0
    )

    length=s[-1]

    new_s=np.linspace(
        0,
        length,
        n
    )

    x=np.interp(
        new_s,
        s,
        points[:,0]
    )

    y=np.interp(
        new_s,
        s,
        points[:,1]
    )

    return np.stack(
        [x,y],
        axis=1
    )
```

# 3. 平滑轮廓

你的情况：

> FSR外形是不规则的，但真实边缘应该是连续的

所以推荐：

## B-spline

```python
from scipy.interpolate import splprep,splev


points=resample_contour(contour)


tck,u=splprep(
    [
        points[:,0],
        points[:,1]
    ],
    s=20,
    per=True
)


u_new=np.linspace(
    0,
    1,
    500
)


x,y=splev(
    u_new,
    tck
)


smooth=np.stack(
    [x,y],
    axis=1
)
```

这里：

```python
per=True
```

很重要。

因为你的 FSR 外轮廓是闭合曲线。

`s`：

控制平滑：

| s  | 效果     |
| -- | ------ |
| 0  | 完全贴合像素 |
| 10 | 轻微平滑   |
| 50 | 明显平滑   |

你的人工mask边缘一般：

```
s=10~100
```

之间调。

# 6. 一个额外建议：不要完全依赖照片坐标

因为你是拍摄：

```
真实FSR
       ↓
相机
       ↓
二维图像
```

存在：

* 透视畸变
* 旋转
* 缩放

如果后续你希望压力显示精确，建议：

拍摄时加入：

* 标定板
* ArUco marker
* 已知尺寸参考物

得到：

[
H
]

单应矩阵：

[
p'=Hp
]

把：

```
图片坐标
```

变成：

```
真实毫米坐标
```

这样以后：

```
sensor spacing = 5mm
```

才有意义。
